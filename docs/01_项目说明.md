# 项目说明

## 1. 项目定位

这是 `D:\agent` 当前 Python 项目的 Java 重构版，目标不是另做一个 Demo，而是在不改变现有产品形态的前提下，把后端替换成：

- `Java 17`
- `Spring Boot`
- `LangChain4j`
- `SQLite`

产品形态仍然保持为：

1. 左侧 2 个业务模块
2. 右侧 1 个智能对话助手
3. 顶部 1 个运行模式与全局状态区

## 2. 当前核心能力

### 2.1 内容创作模块

输入：

- 领域 `field`
- 方向 `direction`
- 想法 `idea`
- 内容分区 `partition`
- 文案风格 `style`

输出：

- 整理后的创作方向 `normalized_profile`
- 当前核心问题 `seed_topic`
- 3 个推荐选题 `topic_result.ideas`
- 推荐主选题 `chosen_topic`
- 一整套文案 `copy_result`

### 2.2 视频分析模块

输入：

- B 站视频链接

输出：

- 自动解析后的视频结构化信息 `resolved`
- 表现判断 `performance`
- 原因分析 `analysis`
- 优化建议 `optimize_result`
- 后续可做选题 `topic_result`
- 可点击参考视频 `reference_videos`
- 低表现时额外返回一套新文案 `copy_result`

### 2.3 智能对话助手

输入：

- 自然语言问题 `message`
- 最近对话历史 `history`
- 页面上下文 `context`

输出：

- 直接回答 `reply`
- 下一步建议 `suggested_next_actions`
- 参考视频 `reference_links`

## 3. Java 版运行逻辑

### 3.1 规则模式

触发条件：

- 页面没有启用运行时 LLM 配置，或者当前模式开关处于关闭状态

特点：

- Web 主工作流走 Java 规则链路
- 智能助手关闭
- 不消耗 token
- 页面顶部显示“无 Key 逻辑模式”

### 3.2 LLM 模式

触发条件：

- 页面模式开关开启，且当前存在一组可用的 LLM 配置
- 这组配置既可以来自 `.env` 启动默认值，也可以来自页面运行时填写

特点：

- 文案生成和聊天通过 `LangChain4j` 调用模型
- 模块创建和模块分析支持 LLM 回退链路
- 会消耗 token
- 页面顶部显示“LLM Agent 模式”

## 4. 当前代码结构

```text
controller/           -> Spring MVC 接口层
service/              -> 业务编排、B 站解析、规则链路、LLM 链路
repository/           -> SQLite 持久化
model/                -> 领域模型
cli/                  -> 命令行入口
src/main/resources    -> application.yml + 前端静态资源
docs/                 -> 项目文档
```

## 5. 当前 Web 接口

### 5.1 主界面核心接口

- `GET /api/runtime-info`

- `POST /api/runtime-llm-config`
- `POST /api/resolve-bili-link`
- `POST /api/module-create`
- `POST /api/module-analyze`
- `POST /api/chat`

### 5.2 兼容接口

- `POST /api/topic`
- `POST /api/copy`
- `POST /api/operate`
- `POST /api/optimize`
- `POST /api/pipeline`

## 6. 数据来源

### 6.1 B 站公开数据

- 视频详情
- 热榜样本
- 分区样本
- 同类 UP 样本
- 当前视频相关推荐
- B 站搜索结果
- 评论数据

### 6.2 本地 SQLite

- 默认文件为 `bilibili_agents.db`
- 主要存储视频指标历史

### 6.3 LLM Provider

- 仅在存在可用运行时配置且页面开关打开后启用
- 通过 `LangChain4j OpenAI Compatible` 调用

## 7. 当前边界

这个项目当前强调：

- 可运行
- 可学习
- 可改造

但它不是：

- 企业级多租户后台
- 高并发生产调度系统
- 完整的商业化账号运营平台
