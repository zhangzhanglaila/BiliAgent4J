# 完整部署文档

## 1. 环境要求

- `Java 17`
- `Maven 3.9+`
- Windows 环境下建议直接使用 PowerShell

## 2. 目录

项目目录：

```text
D:\BiliAgent4J
```

## 3. 配置环境变量

复制示例配置：

```powershell
Copy-Item .env.example .env
```

推荐最少配置：

```env
LLM_PROVIDER=openai
LLM_API_KEY=
LLM_BASE_URL=https://zapi.aicc0.com/v1
LLM_MODEL=gpt-5.4
DB_PATH=bilibili_agents.db
REQUEST_INTERVAL=1.2
DEFAULT_PARTITION=knowledge
DEFAULT_PEER_UPS=546195,15263701,777536
```

说明：

- 如果 `LLM_API_KEY` 为空，系统启动后默认处于规则模式。
- 如果填入 `LLM_API_KEY`，系统启动后会把它当成一组默认可复用的 LLM 配置。
- 页面顶部也支持直接填写运行时 LLM 配置，并立即切换模式，不需要重启服务。

## 4. 启动项目

### 4.1 Web 模式

```powershell
mvn spring-boot:run
```

访问地址：

```text
http://127.0.0.1:8000
```

### 4.2 打包

```powershell
mvn -DskipTests package
```

打包产物位置：

```text
D:\BiliAgent4J\target
```

### 4.3 CLI

```powershell
mvn spring-boot:run "-Dspring-boot.run.arguments=topic --partition=knowledge --topic=AI 剪辑效率"
mvn spring-boot:run "-Dspring-boot.run.arguments=copy --topic=AI 剪辑第一条视频先拍什么更容易起量 --style=干货"
mvn spring-boot:run "-Dspring-boot.run.arguments=optimize --bv=BV1xx411c7mD"
```

## 5. 运行模式说明

### 规则模式

- 不需要 LLM Key
- 不消耗 token
- 聊天助手关闭
- 内容创作和视频分析仍然可用

### LLM Agent 模式

- 需要 `LLM_API_KEY`
- 文案和对话会调用 LangChain4j
- 会消耗 token

### 页面运行时切换

- 页面顶部有运行模式开关
- 如果当前没有已保存的 LLM 配置，打开开关时会先要求填写 `URL / Key / provider / model`
- 保存后立即切到 `LLM Agent` 模式
- 关闭开关后立即回到规则模式，但不会清空当前已保存配置

## 6. 常见问题

### 6.1 页面打不开

检查：

- `mvn spring-boot:run` 是否成功
- `8000` 端口是否被占用

### 6.2 视频解析失败

可能原因：

- 短链未正常展开
- B 站公开接口临时不可用
- 目标链接不包含有效 BV 号

### 6.3 LLM 不可用

检查：

- 页面保存的运行时 `URL / Key / provider / model`
- 或 `.env` 中的默认 `LLM_API_KEY / LLM_BASE_URL / LLM_MODEL`
- 当前运行模式开关是否已打开
