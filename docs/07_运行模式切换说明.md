# 运行模式切换说明

## 1. 当前有两种模式

### 规则模式

触发条件：

- 当前没有启用运行时 LLM 配置
- 或者页面顶部的运行模式开关处于关闭状态

特点：

- 不消耗 token
- 聊天助手关闭
- 选题、文案、视频分析仍然可用

### LLM Agent 模式

触发条件：

- 页面顶部的运行模式开关已开启
- 当前已经存在一组可用的 LLM 配置

特点：

- 聊天助手开启
- 文案与部分分析链路可调用 LangChain4j
- 会消耗 token

## 2. 如何从规则模式切到 LLM 模式

### 方式一：提前在 `.env` 里写默认配置

```text
D:\BiliAgent4J\.env
```

填入：

```env
LLM_API_KEY=your-real-key
LLM_BASE_URL=https://zapi.aicc0.com/v1
LLM_MODEL=gpt-5.4
```

- 启动服务后，页面会自动识别这组默认配置
- 再打开顶部运行模式开关，即可进入 `LLM Agent` 模式

### 方式二：直接在页面填写运行时配置

- 打开页面顶部的运行模式开关
- 如果当前没有已保存配置，页面会展开运行时配置表单
- 填写 `URL / Key / provider / model`
- 点击“保存配置并开启 LLM Agent”

这一种方式会立即生效，不需要重启服务。

## 3. 如何切回规则模式

- 直接关闭页面顶部运行模式开关即可
- 这会立即切回规则模式
- 已保存的运行时配置仍会保留，后面再次打开开关时可以直接复用

如果你希望服务启动时也不再自动带默认配置，再把 `.env` 中的 `LLM_API_KEY` 清空即可。

## 4. 页面上怎么看当前模式

页面顶部会直接显示：

- 当前运行中：无 Key 逻辑模式
- 当前运行中：LLM Agent 模式

## 5. 开发时建议

### 联调接口

优先在规则模式下先跑通。

### 联调 LLM

再切到 LLM 模式看增强行为。

这样更容易定位问题究竟是：

- Java 规则链路问题
- 还是模型调用问题
