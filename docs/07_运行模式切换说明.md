# 运行模式切换说明

## 1. 当前有两种模式

### 规则模式

触发条件：

- `.env` 中 `LLM_API_KEY` 为空

特点：

- 不消耗 token
- 聊天助手关闭
- 选题、文案、视频分析仍然可用

### LLM Agent 模式

触发条件：

- `.env` 中填写了 `LLM_API_KEY`

特点：

- 聊天助手开启
- 文案与部分分析链路可调用 LangChain4j
- 会消耗 token

## 2. 如何从规则模式切到 LLM 模式

编辑：

```text
D:\Agent4J\.env
```

填入：

```env
LLM_API_KEY=your-real-key
LLM_BASE_URL=https://zapi.aicc0.com/v1
LLM_MODEL=gpt-5.4
```

然后重启服务：

```powershell
mvn spring-boot:run
```

## 3. 如何切回规则模式

把 `.env` 中的 `LLM_API_KEY` 清空，然后重启服务。

## 4. 页面上怎么看当前模式

页面顶部会直接显示：

- 当前运行中：无 Key 逻辑模式
- 当前运行中：LLM Agent 模式

## 5. 开发时建议

### 联调接口

优先在规则模式下先跑通。

### 联调 LLM

再切到 LLM 模式看增强行为。

这样更容易定位问题究竟是：

- Java 规则链路问题
- 还是模型调用问题
