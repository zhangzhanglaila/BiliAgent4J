# 前端说明

## 1. 前端当前状态

Java 版前端静态资源沿用了原项目现有页面：

- `index.html`
- `app.js`
- `style.css`

资源目录：

```text
src/main/resources/static
```

## 2. 页面结构

页面仍然分为三部分：

1. 顶部全局状态区
2. 左侧业务模块区
3. 右侧智能对话助手

左侧业务模块通过顶部标签切换：

- 视频分析
- 内容创作

## 3. 前端依赖哪些接口

- `GET /api/runtime-info`
- `POST /api/resolve-bili-link`
- `POST /api/module-create`
- `POST /api/module-analyze`
- `POST /api/chat`

同时也保留兼容接口：

- `POST /api/topic`
- `POST /api/copy`
- `POST /api/operate`
- `POST /api/optimize`
- `POST /api/pipeline`

## 4. 当前交互特性

前端保留了这些行为：

- 粘贴视频链接后自动解析
- 模块切换
- 全局状态提示
- 进度条动画
- 一键复制
- 智能助手输入区
- 语音输入按钮
- 聊天打字机式输出
- 参考视频卡片展示

## 5. 前后端协作边界

前端主要负责：

- 收集输入
- 调接口
- 渲染结果

后端主要负责：

- 真正的视频解析
- 规则判断
- 文案生成
- LLM 调用
- 参考视频筛选

## 6. 改前端时要注意什么

如果你要继续改前端，优先保证：

1. API 路径不乱改
2. JSON 字段尽量向后兼容
3. 结果结构不要随意扁平化

因为当前页面渲染逻辑大量依赖现有字段命名。
