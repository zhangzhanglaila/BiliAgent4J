# 当前项目功能实现原理

## 1. 总体架构

- `ApiController` 暴露 HTTP 接口
- `WorkspaceService` 负责传统模块流程
- `LlmWorkspaceAgentService` 负责 Agent 编排
- `KnowledgeBaseService` 负责本地知识库存储与检索
- `LongTermMemoryService` 负责长期记忆

## 2. 知识库原理

### 2.1 上传

- `KnowledgeFileTextExtractor` 解析文本
- `KnowledgeBaseService` 负责切片、向量化、写入

### 2.2 检索

- 通过本地 deterministic embedding 生成向量
- 结合 lexical overlap 计算排序
- 再做同文档 collapse

### 2.3 热门样本同步

- `KnowledgeSyncService` 抓取热门榜和分区榜
- 把每条视频转成结构化文档
- 写入知识库
- `KnowledgeUpdateJobService` 负责异步任务状态

## 3. 长期记忆原理

- 每次模块执行或对话完成后异步写入
- 下次任务先拉取历史经验
- 再叠加即时检索结果

## 4. Agent 工具原理

当前 Java 版 Agent 已注册工具：

- `retrieval`
- `creator_briefing`
- `video_briefing`
- `hot_board_snapshot`
- `web_search`
- `code_interpreter`

工具调用过程：

1. 组装当前任务 payload
2. 让 LLM 选择下一步动作
3. 执行工具
4. 把 observation 写回 scratchpad
5. 最终输出结构化 JSON

## 5. 前端与后端协同

- 首页加载后先拉取 `runtime-info`
- 知识库页签单独拉取 `knowledge/status`
- 热门同步使用异步任务轮询
- 上传、样本查看、检索都走独立 API
